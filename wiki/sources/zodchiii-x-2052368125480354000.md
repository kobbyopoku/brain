# darkzodchi — Why 90% of AI teams die in 30 days (Teamly product pitch)

> darkzodchi argues that **90% of AI teams die in 30 days not because of agent capability but because of monitoring + infrastructure failures**. The thread is content-marketing for [[wiki/entities/teamly|Teamly]] — a $29-$179/month managed-AI-team platform the author tested against [[wiki/entities/claude-code|Claude Code]], [[wiki/entities/cowork|OpenClaw]], [[wiki/entities/n8n|n8n]], and Render. Key claim: *"where they live is the entire game"* — infrastructure matters more than agent intelligence alone.

## TL;DR

Three rules for AI teams that survive past 30 days:

1. Agents need **specific job descriptions** with narrow, repeatable tasks (not "generalist agents").
2. **Real-time visibility** into agent operations is essential (logs, dashboards, alerts).
3. **Hosting on personal devices is unsustainable** (Mac Mini running 24/7 isn't a solution).

After comparing platforms, darkzodchi recommends **[[wiki/entities/teamly|Teamly]]** — a managed platform with pre-built team templates, a "Pixel Department" monitoring interface, and bundled infrastructure + API management. Pricing: $29-$179/month tiers.

## Key takeaways

### "Where they live is the entire game"

The most quotable claim. Infrastructure matters more than model intelligence. Aligns with [[wiki/sources/Mnilax-430-hours-claude-code-waste|Mnilax's 430-hour study]] of token-waste patterns in Claude Code — the *operational layer* (caching, hook overhead, MCP tool defs, skills loading) determines reliability, not the LLM.

### Cost arithmetic for replacing employees

- **Replacement cost**: $2,000-$4,500/month per role.
- **AI cost**: ~$89 hosting + $700-$900 API spend = ~$789-$989/month per AI team.

This matches CyrilXBT's parallel arithmetic in [[wiki/sources/cyrilxbt-x-2052570518667378918]] (5 agents @ $105/mo + ~$700-$900 API ≈ $800-$1,000/month). **Two independent practitioners converge on the same cost-arithmetic for AI-replacing-employees.** Strengthens existing wiki canon on [[ai-automation-services]].
