# Khairallah — Three-session Cowork architecture (Morning Briefing / Midday Production / End-of-Day Wrap-up)

> Khairallah ([[wiki/entities/eng-khairallah]]) presents a **three-session daily Cowork architecture** that brackets a knowledge worker's entire day: **7AM Morning Briefing** (auto-scheduled), **Midday Production Block** (manually-triggered batch work), **5PM End-of-Day Wrap-up** (auto-scheduled, closes the loop and seeds tomorrow's briefing). Distinguishes *conversational* AI use ("interruption management") from *systems* AI use ("infrastructure you build"). Specific connectors: Gmail / Google Calendar / Slack. Refinement loop: 15 min every Friday. Quote: ***"Claude Cowork is not a tool you use. It is infrastructure you build."***

## TL;DR

Three Cowork sessions per workday:

1. **7AM — Morning Briefing** (scheduled via `/schedule`): scans email + categorizes by 4-tier urgency, drafts routine responses, flags messages that need human brain, pulls calendar + meeting prep briefs, summarizes overnight Slack. Output: one markdown file in `/Documents/Daily-Briefings/`.
2. **Midday — Production Block** (manually triggered): uses pre-built task templates for the user's 5 most-common recurring tasks (document processing, data compilation, content production, research compilation). Cowork operates on actual files, not chat-paste output.
3. **5PM — End-of-Day Wrap-up** (scheduled): reviews day's email + calendar + file modifications, cross-references against the morning briefing to flag what got handled vs what didn't, includes a *carry-forward* section that becomes tomorrow's priorities.

Setup time: ~1-2 days. Daily savings: 1-3 hours = 20-60 hours/month.

## Key takeaways

### Conversational AI vs systems AI

> *"Most people's relationship with AI is a conversation. They open Claude. They ask a question. They get an answer. They close the tab. They come back an hour later and do it again. That is not productivity. That is interruption management."*

This is the strongest articulation in the wiki of the *conversation → system* progression. People who get 10× leverage are not having conversations; **they are running systems**. Worth canonizing as a sub-concept under [[cowork]] / [[claude-code]].

### Cowork operates on actual files

Khairallah's most operationally-significant claim: *"It does not give you text to copy and paste. It creates the documents, updates the spreadsheets, compiles the reports, and saves everything exactly where it belongs."* The chat-then-paste workflow is the bottleneck. Cowork's local-filesystem write capability is what makes the production block possible. Aligns with [[wiki/sources/heygurisingh-x-cowork-setup|Guri Singh's Cowork setup guide]] and the wiki's [[markdown-as-agent-contract]] thesis (the file IS the contract).

### Three-session architecture as DOE composition

Each session is a [[doe-framework|DOE]] cycle:

- **Morning Briefing**: Directive (urgency tiers + briefing format) + Orchestration (scheduled trigger) + Execution (Cowork reads inboxes/calendar/Slack and writes file).
- **Production Block**: Directive (task templates) + Orchestration (manual trigger + parallelized sub-agents) + Execution (file processing).
- **End-of-Day Wrap-up**: Directive (wrap-up format with carry-forward) + Orchestration (scheduled) + Execution (review + write).

The carry-forward field is the **state-passing mechanism** between sessions. It's what makes the daily loop *closed* rather than *episodic*.

### 4-tier email urgency system

Khairallah's explicit tiers (the precision is what makes the agent useful):

- **Tier 1**: needs response before 9AM
- **Tier 2**: needs response today
- **Tier 3**: can wait until this week
- **Tier 4**: informational, no response needed

*"What counts as urgent for your business is different from what counts as urgent for someone else. Define your tiers. The more specific your tiers and categories, the better the briefing."*

This is the same principle as CyrilXBT's voice-profile extraction — **the precision of the directive determines the value of the output**.

### Weekly Friday refinement (15 min)

Khairallah prescribes a 3-question Friday review:

1. *"What did the briefing miss that I had to discover on my own?"* → update the prompt.
2. *"What did Cowork produce that I had to redo?"* → update the template to fix the quality issue.
3. *"What new recurring task appeared this week that I should automate next?"* → build a new template.

This is **manual [[self-annealing]]** — the human is the optimizer, not an Analytics agent (cf. [[wiki/sources/cyrilxbt-x-2052570518667378918|CyrilXBT's automated Analytics agent]]). Both approaches solve the same problem (system improves over time) via different mechanisms — manual review + prompt edit vs autonomous agent rewriting other agents' prompts. **Compose them: Khairallah's 15min Friday + CyrilXBT's monthly Analytics agent = strongest pattern.**

### Production-block task templates

The 4 high-value Cowork production patterns Khairallah names:

| Pattern | What Cowork does |
|---|---|
| **Document processing** | Folder of PDFs/contracts → extract / summarize / organize / compile. Sub-agent parallelization handles 20 docs in minutes. |
| **Data compilation** | Read multiple spreadsheets → combine + calculate → formatted report. |
| **Content production** | Read content brain → produce a week's content, formatted, saved to right folders, ready for review. |
| **Research compilation** | Research question + web search → structured research brief in folder. *"Come back to a finished document instead of twenty open browser tabs."* |

Each template specifies: input source, processing steps, output format, save location.
