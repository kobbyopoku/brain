# noisyb0y1 — ECC hackathon project (38 agents / 156 skills / 72 commands; AgentShield)

> noisyb0y1's reveal of **ECC** — a hackathon-winning Claude Code automation tool with **38 specialized agents, 156 skills, 72 commands**, plus a security-audit component called **AgentShield** with 1,282 tests + 102 security rules. Material because (a) it surfaces **skill-marketplace malware as a real risk** (*"12% of skills on one marketplace were malware as of January 2026"*) and (b) it ships with **persistent agent instincts** that learn coding style across 2-3 weeks of use. New entity stub for [[wiki/entities/ecc|ECC]] created in this batch.

## TL;DR

ECC is a comprehensive Claude Code automation toolkit:

- **38 specialized agents** + **156 skills** + **72 commands** working together.
- **Planner agent** coordinates work across specialists; *"completes day-long junior developer tasks in 20-40 minutes."*
- **12 programming language ecosystems** supported (TypeScript, Python, Go, Rust, others).
- **AgentShield security component**: 1,282 tests + 102 security rules for auditing agent configurations.
- **Persistent learning system**: develops *"instincts"* across sessions — Claude learns the user's coding style and conventions over 2-3 weeks.

Three complementary tools recommended alongside:
- [[wiki/entities/claude-mem|claude-mem]] (38,000+ stars) — cross-session memory.
- [[wiki/entities/superpowers|Superpowers]] — structured planning before coding.
- `CLAUDE.md` rules — consistent agent behavior.

Value claim: *"replaces $8,000-15,000/month development teams at $20/month."*

## Key takeaways

### Skill-marketplace malware is a documented risk

The most consequential claim in the thread: *"12% of skills on one marketplace were malware as of January 2026."* This is a **specific empirical claim about agentic-AI security** that aligns with the wiki's existing concern from [[wiki/sources/NainsiDwiv50980-equipping-agents-for-real-world|NainsiDwiv's]] flagging of skills' security risk surface — but adds a concrete percentage. Worth tracking; AgentShield is the first wiki-documented tool that audits for this.

### AgentShield as the security-of-skills primitive

AgentShield's 1,282 tests + 102 security rules represents the most concrete implementation in the wiki of *"audit agent configurations for malicious patterns."* Sibling pattern to:
- [[wiki/entities/trail-of-bits-claude-code-skills|Trail of Bits's claude-code-skills]] — security skill collection (different shape: skills *about* security vs auditor *of* skills).
- [[anti-ai-slop-machinery]]'s lint-artifact discipline — output-quality auditing rather than security-pattern auditing.

Worth promoting **skill-marketplace-security** as a wiki concept on a future ingest.

### Agent instincts: persistent learning across sessions

ECC's *"instincts"* feature — Claude learns the user's coding style/conventions across 2-3 weeks — is a fresh implementation of the [[self-annealing|self-annealing]] / [[hot-cache|hot-cache]] / [[wiki/entities/hermes-agent|Hermes Agent]] persistent-memory family. **Now four distinct mechanisms** in the wiki for "agents that improve across sessions":
1. External markdown ([[hot-cache|nateherk's `_hot.md`]] / [[llm-wiki-pattern|Karpathy's LLM Wiki]])
2. Directive-encoded ([[self-annealing|Saraev's self-annealing]])
3. Agent-internal state ([[wiki/entities/hermes-agent|Hermes Agent's]] learning loop)
4. **Instinct-based** (ECC's coding-style accumulation) — distinct from the others; user-style-specific rather than task-specific.

Worth tracking the asymmetry: ECC's instincts are *style-of-the-user*, while Hermes's loop is *skills-of-the-task*. They could compose.

### "Planner agent coordinates specialists"

This is a multi-agent-orchestration pattern with explicit named coordinator. Aligns with [[multi-agent-orchestration]] canon and [[wiki/sources/cyrilxbt-x-2052570518667378918|CyrilXBT's 5-agent system]] (with Analytics agent as orchestrator-of-orchestrators). The pattern is converging across multiple practitioners.
