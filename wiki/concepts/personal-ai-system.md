# Personal AI System

> A stack of persistent-context layers that compound so an AI assistant knows the user across sessions.

## Definition

A Personal AI System is the overarching framing of a layered stack — preferences, projects, memory, voice, files, connectors, and schedules — built on top of Claude so that the assistant retains context about the user across sessions. The system's central claim is a "compound effect": each layer makes every other layer more powerful.

## Why it matters

This is the umbrella concept for several of the wiki's Claude-usage layers (memory, projects, voice matching). It frames Claude not as a stateless chatbot but as a persistent, personalized system — directly relevant to Godwin's agentic-architecture and AI-product interests.

## Treatment across sources

- [[wiki/sources/heynavtoor-personal-ai-system-claude]] frames it as a stack of persistent-context layers (preferences + projects + memory + voice + files + connectors + schedules) that compound, with the core claim that each layer makes every other more powerful.

## Sub-claims and details

- The stack comprises preferences, projects, memory, voice, files, connectors, and schedules. [[wiki/sources/heynavtoor-personal-ai-system-claude]]
- The core "compound effect" claim: each layer makes every other layer more powerful. [[wiki/sources/heynavtoor-personal-ai-system-claude]]
- Personal Preferences ("who you are") are global, in contrast to per-project instructions ("what this workspace is for"). [[wiki/sources/heynavtoor-personal-ai-system-claude]]

## Open questions and contradictions

- The compound-effect claim is asserted by the source rather than demonstrated; no measurement is offered.
