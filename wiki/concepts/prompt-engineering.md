# Prompt engineering

> The engineering discipline of structuring the instructions a model receives so it behaves reliably and consistently — not clever wording or "magic words."

## Definition

Prompt engineering is framed across sources as a structured engineering practice rather than a trick. [[wiki/sources/techwithtimm-ai-engineer-roadmap-2026]] (Stage VI item 1) defines it as structuring instructions for reliable, consistent model behavior — explicitly not clever wording. [[wiki/sources/akshay_pachaar-x-anatomy-of-an-agent-harness]] places it as the innermost engineering level — crafting the instructions the model receives — and as a strict subset of context and harness engineering. [[wiki/sources/Shruti_0810-zero-to-ai-engineer-roadmap]] treats it as "an engineering discipline rather than magic words," studied directly from frontier-lab material.

## Why it matters

Prompt engineering is the most immediately reachable lever over model behavior, and the sources converge on de-mystifying it — reframing it from folklore into engineering. It nests inside the broader context/harness-engineering picture that recurs across the wiki's agent material.

## Treatment across sources

- [[wiki/sources/techwithtimm-ai-engineer-roadmap-2026]] frames it as Stage VI item 1; structuring instructions for reliable, consistent model behavior — not clever wording.
- [[wiki/sources/akshay_pachaar-x-anatomy-of-an-agent-harness]] frames it as the innermost engineering level: crafting the instructions the model receives, a strict subset of context and harness engineering.
- [[wiki/sources/Shruti_0810-zero-to-ai-engineer-roadmap]] frames it as Phase 4, studied directly from frontier labs (OpenAI Academy plus Anthropic docs) as "an engineering discipline rather than magic words."

## Sub-claims and details

- It is structuring instructions for reliable, consistent behavior, not clever wording ([[wiki/sources/techwithtimm-ai-engineer-roadmap-2026]]).
- It is the innermost of nested engineering levels and a strict subset of context and harness engineering ([[wiki/sources/akshay_pachaar-x-anatomy-of-an-agent-harness]]).
- Shruti recommends learning it from frontier-lab sources directly — OpenAI Academy and Anthropic docs ([[wiki/sources/Shruti_0810-zero-to-ai-engineer-roadmap]]).
- All three sources explicitly reject the "magic words" / clever-wording framing ([[wiki/sources/techwithtimm-ai-engineer-roadmap-2026]], [[wiki/sources/Shruti_0810-zero-to-ai-engineer-roadmap]]).

## Open questions and contradictions

- Akshay positions prompt engineering as a strict subset of context/harness engineering; the roadmaps treat it as a standalone stage/phase. These are compatible but emphasize different scopes.
