# Voice Style Guide

> The central artifact a personal AI posting agent runs on — a captured specification of an author's writing patterns (not just topics) that the agent uses to generate content in that author's voice.

## Definition

In [[wiki/sources/doublenickk-personal-x-agent-algorithm]] the **Voice Style Guide** is the document the agent consults before producing any post. It is produced by running a Pattern Analysis prompt (in Claude Code) over the author's existing writing *before building anything else*. The source calls it "the most important document your agent will use — everything else is scaffolding around it." It must capture *patterns* — sentence shapes, rhythm, openings, formatting habits — precisely, not merely the topics the author writes about.

## Why it matters

The Voice Style Guide is a concrete, domain-specific instance of the [[markdown-as-agent-contract]] pattern applied to personal-brand content: a markdown artifact the human authors (via assisted analysis) that the agent acts on every run. For Godwin's personal-brand surfaces it is the lever that separates an agent that sounds like the author from generic AI output.

## Treatment across sources

- [[wiki/sources/doublenickk-personal-x-agent-algorithm]] frames it as the agent's central artifact, produced up-front by Pattern Analysis, reviewed periodically to prevent voice drift, and — in Blueprint 03 — rewritten automatically by an Optimization Agent from real engagement metrics.

## Sub-claims and details

- **Produced up-front**: generated by a Pattern Analysis prompt run in Claude Code before building the agent ([[wiki/sources/doublenickk-personal-x-agent-algorithm]]).
- **Patterns over topics**: must capture writing patterns precisely, not just subject matter.
- **Primacy**: described as the single most important document the agent uses; everything else is scaffolding around it.
- **Drift control**: reviewed weekly/monthly to prevent voice drift toward generic AI tone.
- **Self-rewriting variant**: in Blueprint 03 an Optimization Agent rewrites it automatically from observed engagement metrics — a content-domain [[self-annealing]] loop.
