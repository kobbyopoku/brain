# Shann Holmberg (shannholmberg)

> X content creator. **Co-founder of [[wiki/entities/lunar-strategy|Lunar Strategy]]** (deploys Content OS for clients). Author of the wiki's most architecturally detailed content-production framework — **5M impressions in 2 weeks; 11M views and 100K bookmarks in 2 months** from a near-silent X account. Distinguishes from typical content-AI sources by leaning **fewer agents + more workflows + sharper voice profile** over multi-agent specialization. ## Profile

Shann's single ingested thread is the **Content OS** — a 6-folder workspace structure (`/strategy /voice /stores /runs /modules /workflows`) with one *content object* per `runs/active/<slug>/` folder. Every piece of content is an object with a lifecycle (`captured → idea_review → brief_ready → drafting → verification → review → approved → scheduler_ready → scheduled → published → feedback_24h → feedback_72h → learned`).

Four routes (decided at idea gate): ORIGINAL / REPURPOSE / REWRITE / RESEARCH+IDEATE. 12-point bookmarkability rubric (8 = personal bar). 54-pattern master avoid-slop document (8-pattern starter shipped publicly).

Two-model split: **Opus 4.7 = writer (taste), GPT-5.5 = orchestrator (routing)**. *"The job of writing and the job of running the system are different jobs."*

## Positions and claims

- **Bookmarks are the unit of content quality, not likes.** *"A bookmark is a small promise the reader makes to their future self."*
- **Generic AI content sounds like AI; a properly configured voice profile produces content that sounds like you.**
- **Never publish unedited.** *"The system is an accelerator, not autopilot. Used as automation, it decays."*
- **A tight context packet beats a giant context window almost every time.** Writer context packet target: 400-900 tokens.
- **The agent count is not the lever; the knowledge layer feeding the writer is.** *"I built a 4-agent system. It worked, but it was overbuilt. Four agents for what is, structurally, three production steps and a feedback loop."*
- **The viral postmortem is the highest-leverage prompt** because it forces the model into critic role with concrete lines to point at, instead of vague approval.
- **The job of writing and the job of running the system are different jobs that reward different models.**
- **Most people stop at publish; that is where the system starts earning.** Feedback loop > generation loop.
