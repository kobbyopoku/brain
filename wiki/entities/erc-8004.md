# ERC-8004

> Draft Ethereum standard for autonomous-agent identity, reputation, and validation, positioned as the shared trust substrate threading the autonomous-agent stack.

## Profile

ERC-8004 is a draft Ethereum standard published August 2025, authored by contributors from MetaMask, the Ethereum Foundation, Google, and Coinbase. It defines registries for agent identity, reputation, and validation, and sits on top of MCP and A2A rather than competing with them. In [[wiki/sources/awrigh01-technical-stack-autonomous-agents]] it is positioned as the shared trust substrate threading the whole stack.

## Key facts

- **Status**: draft Ethereum standard published August 2025 (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
- **Authors**: contributors from MetaMask, the Ethereum Foundation, Google, and Coinbase (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
- **Identity Registry**: an ERC-721 NFT where each agent owns a token whose URI resolves to a JSON registration file naming its A2A endpoint, MCP endpoint, ENS handle, wallet addresses, and supported trust models (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
- **Reputation Registry**: thin — stores feedback, not scores (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
- **Validation Registry**: unopinionated about validation method (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
- **Metadata**: supports `setMetadata(agentId, key, value)` for arbitrary on-chain metadata (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
- **Adoption**: per AgentProof, 128,000+ agents registered across 24 chains as of early 2026 (per [[wiki/sources/awrigh01-technical-stack-autonomous-agents]]).
