# Claude Projects

> The Claude.ai feature around which [[wiki/sources/AnatoliKopadze-how-to-actually-use-claude-18-steps]] is built — a persistent workspace that carries context across every conversation inside it.

## Profile

A Claude Project is a persistent workspace on Claude.ai in which Claude retains context across every conversation held inside it. Per [[wiki/sources/AnatoliKopadze-how-to-actually-use-claude-18-steps]], a Project combines three durable layers: a knowledge base Claude reads at the start of each conversation, a Custom / Project Instructions field that sets Claude's permanent operating mode for the Project, and shared memory that survives across individual chats. The 18-step guide treats Projects as the foundation for turning Claude from a search-engine-style chatbot into a context-aware thinking partner.

## Key facts

- **Persistent workspace** — a Project keeps context across every conversation inside it (step 1) ([[wiki/sources/AnatoliKopadze-how-to-actually-use-claude-18-steps]]).
- **Knowledge base** — Claude reads it at the start of every conversation in the Project (step 2).
- **Custom / Project Instructions** — a field that becomes Claude's permanent operating mode for that Project (step 3).
- **Cross-chat memory** — starting a fresh chat inside a Project preserves Project memory while dropping prior-chat context (step 13).

## Positions and claims

- **Projects are the unit of durable Claude context** — the guide's central organizing claim is that a Project, not an individual chat, is where persistent context, instructions, and knowledge live. See [[wiki/sources/AnatoliKopadze-how-to-actually-use-claude-18-steps]].
